API

Backward-compatible. The legacy /convert-image endpoint behaves exactly the same as before — omit everything and you still get a 32×32 JSON pixel grid.

GET / POST /convert-image url=<image url> required when no file uploaded file=<multipart upload> POST alternative to url width=1..4096 default 32 height=1..4096 default 32 format=png|jpeg|webp|bmp|gif|json default json mode=crop|stretch|fit default crop simple=true|false nearest-neighbour pixel mode crop_x,crop_y,crop_w,crop_h pre-crop in source pixels

New tool endpoints

Each accepts url= or a multipart file= upload, same as the legacy route. All return the processed image directly.

POST /api/image/pixelate
block=2..128, mode=simple|og. Pixelates without changing the output dimensions.
POST /api/image/invert
Inverts RGB. Alpha preserved.
POST /api/image/mirror
axis=h|v|both.
POST /api/image/rotate
angle=-360..360, expand=true|false.
POST /api/image/format
format=png|jpeg|webp|bmp|gif, quality=1..100.
POST /api/image/remove-bg
tolerance=0..120, feather=0..20. Corner-sample chroma removal.
